# Module Responsibility



## **Module structure**



| **Module**       | **Purpose**                             | **Inputs**                         | **Outputs**                                        | **Owner Suggestion** |
| ---------------- | --------------------------------------- | ---------------------------------- | -------------------------------------------------- | -------------------- |
| main.c           | System startup and foreground loop      | module APIs, status flags          | system orchestration                               | Rene                 |
| sequencer.c/.h   | Core state + sequencing logic           | button/keypad events, BPM updates  | step state, playback state, note-trigger decisions | Rene                 |
| timer_seq.c/.h   | Timer setup and ISR                     | BPM value, play state              | periodic step advance flag / ISR events            | Rene                 |
| midi_uart.c/.h   | UART MIDI transmission                  | note number, velocity, channel     | MIDI byte stream                                   | Rene                 |
| buttons.c/.h     | Read and debounce pushbuttons           | raw GPIO button states             | clean button press events                          | Ignacio              |
| keypad.c/.h      | Scan 4x4 keypad                         | keypad row/column states           | key index 0–15                                     | Ignacio              |
| lcd.c/.h         | LCD initialization and display routines | current track, steps, current step | LCD display output                                 | Shared               |
| display7seg.c/.h | BPM display                             | BPM value                          | 7-segment display output                           | Ignacio              |
| gpio.c/.h        | low-level GPIO init/helpers             | port/pin config needs              | GPIO-ready hardware access                         | Shared / Rene        |



## **What each module “owns”**



### **main.c**

Owns:

- initialization order
- forever loop
- calling input scan/update functions
- acting on flags from ISR
- coordinating UI refreshes

Does not own:

- detailed timer math
- keypad scanning details
- sequencer logic internals

### **sequencer.c**

Owns:

- steps[4] [16]
- current_track
- current_step
- bpm
- playback state
- helper functions like:
  - select track
  - toggle step
  - start playback
  - stop playback
  - advance step
  - get note for track

*This is the application brain.*

### **timer_seq.c**

Owns:

- timer initialization
- timer reload update from BPM
- ISR definition
- lightweight ISR actions

### **midi_uart.c**

Owns:

- UART initialization for MIDI transmission
- sending bytes
- MIDI helper functions like Note On / Note Off

### **buttons.c**

Owns:

- raw button reads
- debounce
- converting electrical press into clean software event

### **keypad.c**

Owns:

- row/column scan logic
- debounce
- key detection

### **lcd.c**

Owns:

- LCD initialization
- writing characters and strings
- display helper functions for sequence drawing

### **display7seg.c**

Owns:

- turning BPM into 4 digits
- digit update / multiplex logic if needed

### **gpio.c**

Owns:

- enabling ports
- base pin configuration helpers
- simple GPIO write/read wrappers if you want them



## **Doxygen blurbs**

### **main.c**

```c
/**
 * @file main.c
 * @brief Main application entry point for the Embedded MIDI Step Sequencer.
 *
 * This file initializes all hardware and software modules used by the
 * sequencer system and runs the foreground loop. The foreground loop is
 * responsible for scanning user inputs, processing non-time-critical events,
 * updating display outputs, and responding to flags generated by the timer
 * interrupt service routine.
 */
```



### **sequencer.c**

```c
/**
 * @file sequencer.c
 * @brief Core sequencer logic and shared application state.
 *
 * This file contains the main data structures and logic used to implement
 * the 4-track, 16-step MIDI step sequencer. It manages the current track,
 * current playback step, tempo, playback state, and step on/off data for
 * all tracks. It also provides helper functions for track selection, step
 * editing, playback control, and note mapping.
 */
```



### **timer_seq.c**

```c
/**
 * @file timer_seq.c
 * @brief Timer configuration and interrupt-based step timing engine.
 *
 * This file configures a hardware timer to generate periodic interrupts
 * based on the sequencer BPM setting. The timer interrupt service routine
 * advances the playback step and signals the rest of the system when
 * real-time sequencer events occur.
 */
```



### **midi_uart.c**

```c
/**
 * @file midi_uart.c
 * @brief UART-based MIDI transmission driver.
 *
 * This file initializes the UART peripheral for MIDI message transmission
 * and provides helper functions for sending raw MIDI bytes and formatted
 * MIDI note messages such as Note On and Note Off events.
 */
```



### **buttons.c**

```c
/**
 * @file buttons.c
 * @brief Pushbutton input handling and debounce logic.
 *
 * This file reads the Tiva LaunchPad and EduBase pushbuttons, applies
 * software debounce where needed, and provides clean button press events
 * to the main application logic.
 */
```



### **keypad.c**

```c
/**
 * @file keypad.c
 * @brief 4x4 keypad scanning and key detection.
 *
 * This file implements the keypad scanning logic for the EduBase 4x4 keypad.
 * It detects key presses, maps them to sequencer step indices, and provides
 * clean step input events to the application.
 */
```



### **lcd.c**

```c
/**
 * @file lcd.c
 * @brief LCD driver and sequencer display helper functions.
 *
 * This file initializes and controls the 16x2 LCD used by the sequencer.
 * It provides helper functions for writing characters and strings and for
 * displaying the current sequencer pattern and playback position.
 */
```



### **display7seg.c**

```c
/**
 * @file display7seg.c
 * @brief 7-segment display driver for BPM visualization.
 *
 * This file controls the 4-digit 7-segment display and provides functions
 * for showing the current tempo value in BPM.
 */
```



### **gpio.c**

```c
/**
 * @file gpio.c
 * @brief Low-level GPIO initialization and digital I/O helper functions.
 *
 * This file contains GPIO configuration code and utility functions used by
 * the sequencer system for LEDs, buttons, keypad lines, and other digital
 * signals connected to the TM4C123G and EduBase V2 board.
 */
```
